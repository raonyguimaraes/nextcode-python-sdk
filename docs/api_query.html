Query Service¶

Service class¶

Service object for interfacing with the GOR Query API.

This class instance is used to communicate with a RESTFul service. Use the execute and execute_template methods to get started. To view past queries you can use the get_queries and get_query methods which return Query objects.

execute(query, nowait=False, persist=None, relations=None, job_type=None, **kw)[source]¶

Execute a gor statement on the server.

Parameters

  • query (str) – gor query string

  • nowait (bool) – if True, return a Query object with PENDING status instead of waiting for query to finish

  • relations (Optional[List[Dict]]) – virtual relations to include with the query

  • persist (Optional[str]) – File path in the project tree to persist the results to (must start with user_data/)

  • job_type (Optional[str]) – Optional job type for routing purposes

Raises

ServerError, MissingRelations

Optional keyword arguments in the form name=data are converted into virtual relations if relations is not explicitly passed in.

Whether nowait is set or not, the serverside method will wait for a maximum of 2 seconds for the query to transition to a completed status. Therefore, most small queries will return in DONE status.

Return type

Query

get_queries(status=None, user_name=None, limit=100, all=False)[source]¶

Get all queries that have been run by the current user in the current project, optionally filtered by status.

Results are returned in reverse chronological order so latest queries are first.

Note that queries are returned in partial state which means they might not be up to date. query.refresh() can be called to force a refresh.

Parameters

  • status (Optional[str]) – Filter queries by status. e.g. DONE RUNNING FAILED.

  • user_name (Optional[str]) – Show queries for another user (only available to admin).

  • limit (int) – Limit the number of queries returned.

  • all (Optional[bool]) – Fetch all queries in all projects (only available to admin).

Return type

Sequence[Query]

Query Object¶

The Query class represents a query model from the RESTFul Query API.

There are several helper methods available here but you can view the raw server response in the raw member.

Note that the object is dynamic and some of the attributes available are not documented here because they are added from the server response.

class nextcode.services.query.query.Query(service, resp=None)[source]¶

A local proxy representing a serverside Gor Query. This object is returned from query methods in the ‘query’ service.

Parameters

  • service – query service handle

  • resp (Optional[Dict]) – json response from query api endpoint

Example usage:

>>> svc = nextcode.Client(profile_name='test').service('query', project='myproject')
>>> query = svc.get_queries(limit=1)[0]
>>> query.status
'DONE'
